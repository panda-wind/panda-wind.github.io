使用 Web Worker 实现文件分片

先搭好架子

对文件进行分片

设置单个分片大小为5MB，用文件总大小除以单个分片大小得出总分片数，注意需要向上取整。

使用 SparkMD5 进行加密

经过测试，200MB的文件需要4.9s。

如何优化

首先，造成耗时的最主要原因是因为这里使用到了 MD5，进行了大量的计算，造成线程阻塞。

所以关键就是要解决线程阻塞的问题。

在 js 中有 Web Woker。

使用 Web Worker

首先使用navigator.hardwareConcurrency获取cpu的线程树，默认是4个线程，多个线程同时开启。

worker.js

经过测试，使用 Web Worker 之后提升到了 0.7s。

最后

这里踩坑的地方是引用 worker.js 时使用的是相对路径，而 worker.js 里引用到了第三放库 spark-md5，引用地址不是相对路径所以一直报错，后来改成相对路径就好了。
