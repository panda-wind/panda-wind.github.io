盘点几种埋点上报方式

常见的埋点上报方式有ajax,img和navigator.sendBeacon三种，下面介绍下这三种方式

基于ajax的埋点上报

因为埋点上报实际上就是对关键数据上报，需要和服务端交互，所以前端可以和后端约定好一个接口通过ajax进行上报。

缺点
通过ajax请求上报难免会遇到跨域问题，而且如果ajax配置不正确还可能会被浏览器拦截，因为这种方式不太推荐。

基于img的埋点上报

上面说到的ajax上报可能会遇到跨域问题，这里我们可以使用一些支持跨域的标签去实现上报功能。

script,link,img就是最好的用于实现上报的标签。

其中推荐使用img标签

script和link的缺陷
因为埋点上报涉及到请求，所以我们需要保证script或link标签的src可以正常请求。但会发现，创建一个script或link标签，如果不挂在在页面上，并不会发起请求。

只有挂载到页面上时才会请求。

对于需要频繁数据上报的埋点而言，这样会反复操作dom对页面渲染性能造成影响，而且加载 js、css 资源还会阻塞页面渲染，所以script和link标签显然并不合适。

img标签的优势
使用img标签加载就不需要挂载到页面上,new Image创建图片后再设置src就可以请求图片。

总结

  1. 使用 img 标签不需要挂载到页面上，避免了反复操作造成对页面性能的影响
  2. img 的加载不会阻塞 html 解析
  3. img 兼容性好
  4. 通常 img 上报使用 gif 图，合法的 gif 只需要 43 个字节

基于Navigator.sendBeacon的埋点上报

Navigator.sendBeacon 方法可通过 HTTP POST 将少量数据异步传输到服务器上。

优势

  1. 相较于 img 标签，使用 Navigator.sendBeacon 会更规范，数据传输上传输类型更多
  2. 异步的，不会影响到下一个页面的跳转，能保证数据发送出去
  3. 支持跨域

总结

  1. 埋点上报总共有 ajax、img、Navigator.sendBeacon 三种方式， 其中不推荐使用 ajax
  2. img 的兼容性最好
  3. 最合适的方案是Navigator.sendBeacon，不仅支持跨域，且是异步，不会影响页面卸载，作为浏览器的任务，可以保证将数据发送出去。
